10/*
11 Heaplet: a dynamic memory manager operating over a fixed buffer.
12
13 Every possible allocated block has a well-known, unique name in the form of an integral tag. All
14 operations target a specific block, by tag. Blocks are automatically moved around the buffer such that
15 all allocated blocks are kept contiguous at all times; there is no fragmentation in a heaplet. Accessing
16 an allocated block therefore requires an additional operation, to get a current pointer to it.
17 Unallocated blocks consume no space in the heaplet.
18
19 Tag values are significant in that blocks are kept sorted; blocks with lower tags are at lower addresses
20 in memory than blocks with higher tags. This guarantees that when a block with a higher tag is resized,
21 blocks with a lower tag will not move. Used judiciously, this arrangement can minimize the impact of the
22 automatic memory contiguity in the heaplet, and for example allow a piece of code to build data structures
23 inside blocks with higher tags that contain pointers to data in blocks with lower tags.
24
25 Note that 0 is a valid size for allocated blocks. This avoids workarounds in client code when a block size
26 is calculated as 0.
27
28 Supported operations on the heaplet are
29 - Set the size of a block, which encompasses both allocating blocks and changing the size of blocks
30 already allocated. This operation causes movement of all allocated blocks with tags higher than
31 that of the subject block.
32 - Free a block. This operation causes movement of all remaining allocated blocks with tags higher than
33 that of the subject block.
34 - Access a block. Returns a pointer to the block's current address. It is generally unsafe to cache
35 pointers from this operation across set-size and free operations except as described above for those
36 operations.
37 - Get the current size of a block. This helps eliminate the need to track sizes in client code.
38
39 Allocated blocks are kept aligned in memory such that they are suitable to contain any kind of data
40 or structures.
41
42 The design of the heaplet deliberately trades away speed for the ability to maximize memory efficiency.
43 Keeping allocated blocks contiguous in memory, and the extra level of indirection required in order to
44 access blocks in the heaplet, are relatively expensive features. However minimizing memory use is more
45 important than the processing efficiency lost maintaining the heaplet.
46
47 The heaplet is a singleton. Functions declared in this header file operate on the single instance.
48 */

74// Allocates or resizes a block.
75// If the block is currently allocated, it is resized.
76// If the block is currently unallocated, it is allocated.
77// Note that zero is a valid size for an allocated block, and a block with zero size is distinct from an
78// unallocated block.
79// Returns a pointer to the block, or NULL if there's not enough space to alloc/resize.
80// Cached pointers to blocks with a higher tag than the block being allocated/resized are invalid after
81// calling this routine, unless NULL is returned.
82void* HeapletSetSize(HeapletTag_t tag, size_t newSize);
83
84// Frees a block.
85// Cached pointers to blocks with a higher tag than the block being freed are invalid after calling
86// this routine, unless the block being freed is not currently allocated.
87void HeapletFree(HeapletTag_t tag);
88
89// Returns the size of a block. The returned size is the size as most recently specified to
90// HeapletSetSize (not the aligned size).
91// This routine cannot be used to distinguish between allocated and unallocated blocks. If zero is returned,
92// the block is either unallocated, or allocated at 0 size.
93size_t HeapletGetSize(HeapletTag_t tag);
94
95// Returns the current address of an allocated block.
96// Exercise care when caching, as block addresses can change as blocks are allocated/resized and freed.
97// A return value of NULL means the block is currently unallocated.
98void* HeapletAccess(HeapletTag_t tag);

100bool HeapletGetBlock(HeapletTag_t tag, void** ppBlock, size_t* pSize);

102// Sets the size of a block to the maximum possible size, e.g. causes a block to be allocated such that
